| Requirement       | Data Structure  |
| ----------------- | --------------- |
| Need count        | HashMap / Array |
| Need order        | Queue           |
| Need last element | Stack           |
| Need fast lookup  | HashMap         |
| Need sorted order | TreeMap         |


My rule of thumb
Whenever I solve a new problem, I ask these questions in order:

Am I checking whether an element exists? → HashSet
Am I counting frequencies or storing values with keys? → HashMap
Am I using nested loops only to search? → Replace with HashSet/HashMap
Does the problem mention duplicate, unique, distinct, or repeated? → Think HashSet
Do I need the index or count of an element? → Think HashMap.
